# pdfnative-cli v1.0.0

_Released 2026-06-30_

**v1.0.0** is the first stable release. It completes the **verify-side
Long-Term Validation (LTV)** story ŌĆö full RFC 3161 timestamp validation
(PAdES-T) plus OCSP (RFC 6960) and CRL (RFC 5280) revocation checking,
offline by default with opt-in, SSRF-guarded online fetching. The CLI grows
up to top-tier 2026 ergonomics: a `batch` command, shell `completion`
scripts, a `.pdfnativerc.json` config file, and global `--quiet` /
`--no-color` / `--version --json` flags. The two remaining upstream
workarounds are gone ŌĆö now fixed in **pdfnative 1.2.0**.

100 % backward-compatible with v0.3.0. Every v0.3.0 invocation continues to
work unchanged.

## Highlights

- **Verify-side LTV is complete.** `verify` now cryptographically validates
  RFC 3161 timestamp tokens (PAdES-T) and checks OCSP + CRL revocation ŌĆö
  embedded from the PDF `/DSS` offline, or fetched online behind an SSRF
  guard.
- **Offline by default, safe online.** Online revocation only runs through a
  scheme allow-list + private/loopback/link-local/CGNAT/multicast IP blocking
  (IPv4 + IPv6), no redirects, with timeout and size caps.
- **`batch` command.** Render an entire directory of JSON in parallel,
  reusing the full `render` pipeline.
- **`completion` command.** First-class bash / zsh / fish completion scripts.
- **`.pdfnativerc.json` config file.** Project-level default flags, global and
  per-command, discovered cwd-upward.
- **Smart tables.** `--table-wrap`, `--repeat-header`, `--zebra`,
  `--min-row-height`, `--cell-padding`, plus page-by-page streaming for TOC /
  `{pages}` support.
- **Two upstream workarounds removed** ŌĆö `cert-fix` and the local
  signature-placeholder injector, now fixed in pdfnative 1.2.0.

## What's new

### `verify` ŌĆö Long-Term Validation (LTV)

- **RFC 3161 timestamp validation (PAdES-T).** The signature-timestamp-token
  unsigned attribute is now cryptographically validated: the TSA SignerInfo
  signature, the eContent (`TSTInfo`) `messageDigest`, and the
  `messageImprint` binding to the document signature are all checked, and the
  TSA certificate chain is built and trust-evaluated. Reported as
  `timestampValid`, `timestampTime` (`genTime`) and `tsaSubject`. Replaces the
  presence-only `timestampPresent` flag from 0.3.0 (still reported for
  back-compat).
- **OCSP revocation (RFC 6960).** Parses OCSP responses embedded in the PDF
  `/DSS` (PAdES-LT) and, with `--revocation online`, fetches from the
  certificate's AIA OCSP URL. The responder signature is verified and the
  CertID matched before the status is read.
- **CRL revocation (RFC 5280).** Parses CRLs embedded in `/DSS` and, online,
  fetches from the CRL Distribution Point. The CRL signature is verified
  against the issuer before the signer serial is checked.
- **`--revocation offline|online|disabled`** (default `offline`) and
  **`--revocation-policy soft-fail|strict`** (default `soft-fail`). New report
  fields: `revocationChecked`, `revocationStatus`, `revocationSource`,
  `revocationMethod`, `revocationRevokedAt`.
- **SSRF-guarded online fetching.** Opt-in online OCSP/CRL requests pass a
  guard enforcing an http(s) scheme allow-list, DNS resolution with
  private/loopback/link-local/CGNAT/multicast (IPv4 + IPv6) blocking, no
  redirect following, a 10 s timeout and a 5 MiB response cap. Unverifiable
  revocation data yields `unknown`, never `good`.

### New commands & CLI ergonomics

- **`batch`** ŌĆö render every `*.json` file in a directory to PDF in parallel,
  reusing the full `render` pipeline. `--input-dir`, `--output-dir`,
  `--concurrency` (default 4), `--fail-fast`, text/JSON summary; exit 1 if any
  file fails.
- **`completion bash|zsh|fish`** ŌĆö emit a shell-completion script.
- **`.pdfnativerc.json`** config file ŌĆö discovered cwd-upward, with global and
  per-command sections; `--config <file>` / `--no-config`. Precedence is
  **CLI flags > env > config**.
- **Global flags** ŌĆö `--quiet` / `-q`, `--no-color` (+ `NO_COLOR`), and
  `--version --json`.

### `render` ŌĆö pdfnative 1.2.0 features

- **Smart tables** ŌĆö `--table-wrap auto|always|never`, `--repeat-header`,
  `--zebra`, `--min-row-height`, `--cell-padding` fill any `TableBlock` fields
  left unset in JSON (block-level JSON wins). Also available through
  `--layout`.
- **Page-by-page streaming** ŌĆö `--stream-page-by-page` streams at PDF object
  boundaries *after* assembling the document, so TOC blocks and `{pages}`
  placeholders are supported (unlike single-pass `--stream`).
- **PDF/A targets** are now sourced from pdfnative's
  `PDF_A_CONFORMANCE_TARGETS` constant.

### `sign`

- **`--timestamp <tsa-url>`** flag reserved for PAdES-T timestamping.
  Embedding a timestamp token at signing time requires upstream pdfnative
  support; the flag validates the URL and errors clearly today. Timestamp
  *validation* already works via `verify`.

## Try it

### Verify with revocation checking

```sh
# Offline (default): use only timestamps + OCSP/CRL embedded in the PDF /DSS
npx pdfnative verify -i signed.pdf

# Online, strict: fetch OCSP/CRL and fail the signature if revoked/unknown
npx pdfnative verify -i signed.pdf \
  --revocation online --revocation-policy strict
```

Sample `verify` output (truncated):

```json
{
  "signatures": [
    {
      "integrity": true,
      "signatureValid": true,
      "signatureAlgorithm": "rsa-sha256",
      "chainValid": true,
      "trustedRoot": true,
      "timestampValid": true,
      "timestampTime": "2026-06-30T12:00:00Z",
      "tsaSubject": "CN=Demo TSA, O=pdfnative-cli",
      "revocationChecked": true,
      "revocationStatus": "good",
      "revocationSource": "dss",
      "revocationMethod": "ocsp"
    }
  ]
}
```

### Batch-render a directory

```sh
npx pdfnative batch \
  --input-dir  ./invoices/json \
  --output-dir ./invoices/pdf \
  --concurrency 8 --compress
```

### Install shell completion

```sh
# bash (system-wide)
npx pdfnative completion bash | sudo tee /etc/bash_completion.d/pdfnative >/dev/null
# zsh (first fpath entry)
npx pdfnative completion zsh > "${fpath[1]}/_pdfnative"
# fish
npx pdfnative completion fish > ~/.config/fish/completions/pdfnative.fish
```

### Project config file

```jsonc
// .pdfnativerc.json ŌĆö discovered cwd-upward
{
  "render": { "page-size": "letter", "compress": true, "zebra": true },
  "verify": { "revocation": "offline", "revocation-policy": "soft-fail" },
  "batch":  { "concurrency": 8 }
}
```

- **Node:** Ōēź 20.
- **Bun, Deno:** unchanged (`node dist/cli.cjs`).

## Out of scope (upstream-coordinated)

- **Sign-side LTV** ŌĆö embedding a TSA timestamp into the CMS (PAdES-T) and
  writing the DSS dictionary at signing time. This is PDF-writing logic that
  belongs in `pdfnative`; 1.2.0 does not yet expose it. `sign --timestamp` is
  reserved and errors clearly. See [SECURITY.md](../SECURITY.md#network-access--revocation-checking)
  and [ROADMAP.md](../ROADMAP.md).
- PAdES-LT / LTA *production* (full long-term archival signatures).
