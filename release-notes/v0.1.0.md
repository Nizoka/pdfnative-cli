# pdfnative-cli v0.1.0

<!-- GitHub Release title: v0.1.0 — Official CLI for pdfnative (render JSON → PDF, sign, inspect) -->

_Released 2026-04-27_

**v0.1.0** is the inaugural release of the official command-line interface for
the `pdfnative` library. The CLI exposes three core workflows as composable
commands — `render` (JSON document → PDF bytes), `sign` (CMS/PKCS#7 digital
signatures), and `inspect` (PDF metadata extraction) — with zero extra runtime
dependencies. All PDF logic is delegated to `pdfnative`.

100 % backward-compatible with future v0.x releases (CLI is new; compatibility
guarantees apply to the `pdfnative` library version pinning at `^1.0.4`).

## Highlights

- **Three production-ready commands** — `render` (with streaming & PDF/A modes), 
  `sign` (RSA & ECDSA), `inspect` (JSON and text output formats)
- **Zero extra dependencies** — `pdfnative` is the sole runtime dependency
- **23 comprehensive samples** — covering document types, barcodes, tables, forms,
  watermarks, PDF/A conformance, and digital signatures
- **Secret-safe signing** — private keys loaded from environment variables
  (`PDFNATIVE_SIGN_KEY`, `PDFNATIVE_SIGN_CERT`), never logged
- **Stdin/stdout by default** — every command reads from stdin and writes to
  stdout, enabling composable shell pipelines
- **Streaming output** — `render --stream` emits PDF chunks progressively
  for large documents (>50 MB)
- **NPM provenance** — signed builds via GitHub Actions OIDC (SLSA Level 2+)

## Added

### Commands

- **`render` command** — transforms a `DocumentParams` JSON document (file or stdin)
  into a PDF. Flags: `--output file.pdf` (default: stdout), `--stream` (AsyncGenerator
  streaming mode), `--conformance [pdfa1b|pdfa2b|pdfa3b]` (PDF/A compliance).
  ([src/commands/render.ts](../src/commands/render.ts))

- **`sign` command** — applies CMS/PKCS#7 digital signatures to an existing PDF.
  Private key and certificate loaded from environment variables
  (`PDFNATIVE_SIGN_KEY` = base64-encoded private key, `PDFNATIVE_SIGN_CERT` = certificate,
  recommended) or file paths (`--key`, `--cert`; env vars take precedence). Keys are
  never logged. ([src/commands/sign.ts](../src/commands/sign.ts))

- **`inspect` command** — extracts and reports PDF metadata: version, page count,
  encryption type, PDF/A conformance level, signature count, embedded metadata.
  Flags: `--format [json|text]` (default: JSON). ([src/commands/inspect.ts](../src/commands/inspect.ts))

### Utilities & Infrastructure

- **Zero-dep argument parser** (`src/utils/args.ts`) — handles `--flag value`,
  `--flag=value`, `-f value`, boolean flags, `--` pass-through. No third-party
  dependency. Properly rejects unknown flags and validates positional arguments.

- **I/O helpers** (`src/utils/io.ts`) — `readStdin()`, `readFileOrStdin()`,
  `writeOutput()` (streams binary data to stdout or file). All paths validated
  against traversal attacks before filesystem access.

- **CLI error handling** (`src/utils/error.ts`) — `CliError` class with exit codes
  (0 = success, 1 = runtime error, 2 = usage error), formatted output to stderr.

- **Core bridge** (`src/core-bridge/index.ts`) — selective re-exports from `pdfnative`
  (`buildDocumentPDFBytes`, `streamDocumentPdf`, `signPdfBytes`, `PdfReader`).
  Prevents accidental over-exposure of internal APIs.

### Security & Validation

- **Path traversal validation** — all file path arguments (render input, sign input,
  key/cert paths) validated with `path.resolve()` and whitelist checks before
  `fs.readFile()` / `fs.writeFile()`.

- **50 MB JSON input cap** — enforced before `JSON.parse()` to prevent memory exhaustion
  on untrusted input.

- **Global flags** — `--help` / `-h` (exit code 0), `--version` / `-V` (exit code 0),
  `--` (end-of-flags).

## Usage Examples

### Render JSON to PDF (stdin/stdout)

```bash
cat render-doc.json | pdfnative render > output.pdf
```

### Render with streaming (large documents)

```bash
pdfnative render --input huge-doc.json --output huge-doc.pdf --stream
```

### Render with PDF/A-2b conformance

```bash
pdfnative render --input document.json --output archived.pdf --conformance pdfa2b
```

### Sign a PDF

```bash
export PDFNATIVE_SIGN_KEY=$(cat private-key.pem | base64)
export PDFNATIVE_SIGN_CERT=$(cat certificate.crt | base64)
pdfnative sign --input unsigned.pdf --output signed.pdf
```

### Inspect PDF metadata (JSON)

```bash
pdfnative inspect --input sample.pdf --format json | jq .
```

### Inspect PDF metadata (human-readable text)

```bash
pdfnative inspect --input sample.pdf --format text
```

## Known Limitations

- **Watermark centering** — watermark vertical centering has a known limitation in `pdfnative`
  (identified in v1.0.4 testing; tracked for fix in future `pdfnative` release)
- **CLI JSON schema** — the `render` command accepts the full `pdfnative` `DocumentParams` schema;
  some advanced features (custom page sizes, encryption, headers/footers via code) are not yet
  exposed via JSON. Use the `pdfnative` Node.js library directly for these features.
- **PDF encryption** — not yet exposed via CLI; use Node.js library API directly

## Backward Compatibility

This is v0.1.0. Pre-v1.0 CLI releases may introduce breaking changes to the command interface or
flag names without a MAJOR version bump (following the Node.js CLI convention of semantic versioning
for the binary, not the library). Once v1.0.0 is released, full SemVer guarantees apply.

The underlying `pdfnative` library (v1.0.4, pinned via `^1.0.4`) is stable and backward-compatible.
