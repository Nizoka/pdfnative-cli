# pdfnative-cli v0.2.0

<!-- GitHub Release title: v0.2.0 — Full pdfnative coverage (encryption, watermarks, headers/footers, attachments, multilang, verify command) -->

_Released 2026-04-28_

**v0.2.0** lifts pdfnative-cli from "three commands" to a complete, production-grade
front-end for the entire `pdfnative` v1.0.5 stable surface — without breaking a single
v0.1.0 invocation. Render now exposes encryption, watermarks, headers/footers, PDF/A-3
attachments, bundled multi-language fonts, and a table-centric variant. Sign gains
metadata and certificate chains. Inspect gains verbose / per-page / `--check` modes.
A new `verify` command verifies signature integrity, certificate chains, and trust roots.

100 % backward-compatible with v0.1.0. The deprecated `--conformance` flag still works
(with a one-line stderr notice) and will be removed in v1.0.0.

## Highlights

- **`render` — complete layout coverage** via the hybrid `flags + --layout file` model:
  encryption (AES-128/256), watermarks (text & image), headers/footers with placeholders,
  PDF/A-3 file attachments, custom page sizes & margins, compression, multilingual fonts
  (`--lang th,ja`), table variant (`--variant table`), full `PdfLayoutOptions` via
  `--layout file.json`.
- **`sign` — production-ready signing metadata**: `--reason`, `--name`, `--location`,
  `--contact`, `--signing-time`, repeatable `--cert-chain`. ECDSA flag wired (currently a
  clean stub — RSA continues to work; ECDSA tracked for v0.3.0).
- **`inspect` — observability for CI**: `--verbose` (trailer & catalog & XMP),
  `--pages` (per-page metadata), `--check pdfa|signed|encrypted` (CI-friendly exit code).
- **`verify` (NEW)** — verify integrity, certificate chains, and trust roots of every
  CMS/PKCS#7 signature in a PDF. JSON & text output, `--strict` mode, `--trust <root.pem>`
  (repeatable).
- **Zero new runtime dependencies** — `pdfnative ^1.0.5` remains the sole dependency;
  every new feature delegates to it.
- **Security-by-default** — encryption passwords + PEM material redacted from all error
  output; layout-file `attachments[].data` injection blocked; path traversal validated on
  every file argument.

## Security

- **feat(security):** PEM redaction guarantee extended to all key paths
  (`--key`, `--cert`, `--cert-chain`, `--trust`, env equivalents). `loadPem` and
  `loadPemChain` only emit generic error messages on parse failure.
- **feat(security):** encryption passwords honour env-var precedence
  (`PDFNATIVE_ENCRYPT_OWNER_PASS`, `PDFNATIVE_ENCRYPT_USER_PASS`); passwords never logged.
- **feat(security):** `--layout` JSON loader strips `attachments[].data` fields on load
  to prevent embedding arbitrary bytes from a JSON file (binary payloads must come from
  `--attachment <path>`, where the path is traversal-validated).
- **feat(security):** `--encrypt-*` rejected when combined with `--tagged pdfa*` per
  ISO 19005 (exit 2 with explicit error).

## Added

### `render` command

- **feat(render):** `--variant document|table` selects `buildDocumentPDFBytes` (default)
  or `buildPDFBytes` (`PdfParams` shape).
- **feat(render):** `--page-size` with named sizes (`a4`, `letter`, `legal`, `a3`,
  `tabloid`, `a5`) or `WxH` in points; `--margin <N>` or `--margin <t,r,b,l>`;
  `--compress` boolean.
- **feat(render):** `--tagged <none|pdfa1b|pdfa2b|pdfa2u|pdfa3b>` unified PDF/A flag.
- **feat(render):** watermark flags — `--watermark-text`, `--watermark-image`,
  `--watermark-opacity`, `--watermark-angle`, `--watermark-color`, `--watermark-font-size`,
  `--watermark-position background|foreground`.
- **feat(render):** header / footer flags — `--header-{left,center,right}`,
  `--footer-{left,center,right}` with `{page}`, `{pages}`, `{date}`, `{title}` placeholders.
- **feat(render):** encryption flags — `--encrypt-owner-pass`, `--encrypt-user-pass`,
  `--encrypt-algorithm aes128|aes256` (default `aes128`),
  `--encrypt-permissions print,copy,modify,extractText`.
- **feat(render):** `--attachment <path>[:mime[:relationship[:description]]]`, repeatable.
- **feat(render):** `--lang <code,code>` loads bundled Noto fonts via `loadFontData`.
- **feat(render):** `--layout <file.json>` loads a `Partial<PdfLayoutOptions>`.

### `sign` command

- **feat(sign):** `--algorithm rsa-sha256|ecdsa-sha256` (RSA works; ECDSA stubbed cleanly).
- **feat(sign):** `--reason`, `--name`, `--location`, `--contact`, `--signing-time`.
- **feat(sign):** `--cert-chain <path>` (repeatable) and `PDFNATIVE_SIGN_CHAIN` env var.

### `inspect` command

- **feat(inspect):** `--verbose` adds trailer keys, catalog keys, object count, XMP.
- **feat(inspect):** `--pages` adds per-page metadata array.
- **feat(inspect):** `--check pdfa|signed|encrypted`, repeatable, ANDed.

### `verify` command (NEW)

- **feat(verify):** verifies byte-range integrity (SHA-256), certificate chain signatures,
  and trust evaluation. JSON & text formats. `--strict` for CI assertions.

### Samples & infrastructure

- **feat(samples):** five new render categories (`encryption`, `headers-footers`,

- **feat(samples):** `samples/run-all.js` extended with per-category flag dispatch and
  per-file overrides (multilang).
- **chore(deps):** `pdfnative` bumped from `^1.0.4` to `^1.0.5`.
- **chore(meta):** `src/utils/layout.ts`, `src/utils/keys.ts`, `getStringFlagAll`,

## Upgrade from v0.1.0

No breaking changes. Drop-in replacement.

| If you are using…                          | Action                                                |
|--------------------------------------------|-------------------------------------------------------|
| `pdfnative render --conformance 2b …`      | _Optional_: switch to `--tagged pdfa2b` (the older flag still works with a stderr notice). |
| `pdfnative sign --key … --cert …`          | No change required; new flags are additive.           |
| `pdfnative inspect --format json …`        | No change required; existing fields preserved, new ones additive. |
| Wanting per-page metadata                  | Add `--pages` to your existing inspect call.          |
| Wanting CI-friendly PDF/A assertion        | Use `pdfnative inspect --check pdfa` (exit 0 / 1).    |
| Wanting to verify signatures on a PDF      | Use the new `pdfnative verify --input … --strict`.    |

## Known limitations

- **ECDSA signing** is stubbed. The `--algorithm ecdsa-sha256` flag is parsed but
  pdfnative 1.0.5 does not yet expose `parseEcPrivateKey`. Tracked for v0.3.0.
- **`verify`** scope is integrity + chain + trust. Full CMS-signature-value verification,
  OCSP / CRL revocation, and RFC 3161 timestamp validation are deferred to v0.3.0+.
- **`{pages}` placeholder** in headers/footers is incompatible with `--stream`
  (multi-pass pagination required); rejected up-front with exit 2.
- **PDF/A + encryption** are mutually exclusive per ISO 19005; rejected up-front with
  exit 2.
