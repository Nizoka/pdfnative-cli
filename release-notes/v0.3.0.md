# pdfnative-cli v0.3.0

<!-- GitHub Release title: v0.3.0 — Full Sign & Verify (ECDSA, CMS, RFC 3161) + render --watch / --template / --font -->

_Released 2026-05-05_

**v0.3.0** finishes the digital-signature story: ECDSA-SHA256 signing,
end-to-end CMS / PKCS#7 cryptographic verification (RSA + ECDSA), RFC 3161
timestamp token recognition, and an automatic signature-placeholder injector
that lets you sign any `pdfnative render` output with a single command — no
hand-built AcroForm required. `render` gains a watch loop, JSON template
merging, and an explicit allow-list of bundled font shortcuts.

100 % backward-compatible with v0.2.0. Every v0.2.0 invocation continues to
work unchanged.

> zero-dependency PDF engine that powers this CLI. Every star helps the
> long-term project.

## Highlights

- **`sign` is now an end-to-end pipeline.** Sign a freshly-rendered PDF in
  one command — the missing AcroForm placeholder is injected automatically.
- **ECDSA-SHA256 is fully wired.** `--algorithm ecdsa-sha256` accepts SEC1
  P-256 keys directly; no extra setup.
- **`verify` does real crypto.** Signature value, message digest, certificate
  chain, trust root, *and* RFC 3161 token presence — all in a single JSON
  report.
- **`render --watch` for fast iteration**, `render --template` for layered
  JSON inputs, `render --font latin|emoji` for one-flag font activation.

## What's new

### `sign` — full end-to-end signing pipeline

- **ECDSA-SHA256 signatures.** `--algorithm ecdsa-sha256` is fully wired;
  loads SEC1 / PKCS#8 P-256 keys via `parseEcPrivateKey`. RSA-SHA256 stays
  the default.
- **Automatic signature placeholder injection.** The CLI detects PDFs with
  no AcroForm signature field and performs a single incremental update
  adding `/Sig`, the signature widget, and `AcroForm /SigFlags 3`. PDFs
  that already carry a placeholder are signed unchanged.
- **`ensureCryptoReady()`** — pdfnative's async ASN.1 module is now booted
  transparently on first use (was a footgun in 0.2.0).

### `verify` — full CMS / PKCS#7 cryptographic verification

- **Signature value verification (RSA-SHA256, ECDSA-SHA256).** Returned as
  `signatureValid` + `signatureAlgorithm` in the JSON report.
- **RFC 3161 timestamp token recognition.** Reported as `timestampPresent`.
  Full TSA-chain validation tracked for v0.4.0.
- **Robust ASN.1 walker.** Replaces the relative-offset bug in
  `pdfnative.derDecode`, plus a re-slicing workaround for
  `cert.issuer.raw` / `cert.subject.raw`. Chain building and self-signed
  detection are correct again for every embedded certificate.

### `render` — iteration & template ergonomics

- **`--watch`** — re-render on input file change (200 ms debounce, stderr-
  only logs, requires `--input <file>` and a file `--output`).
- **`--template <file.json>`** — deep-merge a base template under stdin /
  `--input`. Plain objects merge recursively; arrays and primitives are
  replaced (caller wins).
- **`--font <name>`** — register a bundled pdfnative font shortcut.
  Repeatable; allow-list of `latin` (Noto Sans VF) and `emoji` (Noto
  Emoji). After registration the name is usable through `--lang`.

## Try it

### Sign and verify a freshly-rendered PDF

```sh
# 1. Render
echo '{"blocks":[{"type":"paragraph","text":"Hello, signed world."}]}' \
  | npx pdfnative render -o out.pdf

# 2. Sign (RSA or ECDSA)
npx pdfnative sign -i out.pdf -o signed.pdf \
  --key tests/fixtures/rsa-key.pem \
  --cert tests/fixtures/rsa-cert.pem \
  --reason "Demo"

# 3. Verify
npx pdfnative verify -i signed.pdf
```

Sample `verify` output:

```json
{
  "signatures": [
    {
      "integrity": true,
      "signatureValid": true,
      "signatureAlgorithm": "rsa-sha256",
      "chainValid": true,
      "trustedRoot": true,
      "timestampPresent": false,
      "signerSubject": "CN=pdfnative-cli RSA Test, O=pdfnative-cli, C=FR",
      "signerIssuer": "CN=pdfnative-cli RSA Test, O=pdfnative-cli, C=FR",
      "notes": ["no --trust provided; accepted self-signed root"]
    }
  ]
}
```

### Watch a file

```sh
npx pdfnative render --input report.json --output report.pdf --watch
# edit report.json — pdfnative-cli re-renders on every save (200ms debounce).
```

### Template + override

```sh
# template.json carries title, layout, headers; stdin overrides the body
echo '{"blocks":[{"type":"paragraph","text":"Today only."}]}' \
  | npx pdfnative render --template template.json -o today.pdf
```

### Bundled fonts

```sh
echo '{"blocks":[{"type":"paragraph","text":"Hi 🚀"}]}' \
  | npx pdfnative render --font latin --font emoji --lang latin,emoji -o out.pdf
```

## Compatibility

- **`pdfnative` peer:** `^1.1.0` (was `^1.0.5`).
- **Node:** ≥ 20.
- **Bun, Deno:** unchanged.
- **No breaking changes from v0.2.0.**

## Out of scope (tracked for v0.4.0)

- Full RFC 3161 timestamp validation (TSA chain, MD compare).
- OCSP / CRL revocation checks.
- LTV (Long-Term Validation) PAdES-LTA.
- Custom (non-bundled) font registration via `--font name=<path>`.
